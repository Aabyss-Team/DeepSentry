# DeepSentry 操作手册

> 本文档与 [README.md](../README.md) 同步维护，偏向“怎么用”；README 偏向总览、配置和开发说明。

> AI 安全应急 Agent · 控制端 + 远程目标机双视角 · 类 Claude Code TUI

## 当前版本重点变化

| 能力 | 用户该怎么理解 |
|------|----------------|
| 初始化三种模式 | 首次 `--init` 可选择本地、远程 1 台、远程多台 Fleet |
| Shell-first 执行 | 默认优先用目标机原生 Shell；内置工具用于 Shell 不够稳、需要结构化解析或控制端探测的场景 |
| 多目标 Subagent | 对 `targets[]` 批量任务时，每台服务器使用独立连接上下文，避免串目标 |
| TUI 输入优化 | 输入行固定在最后一行，改善 macOS/Windows 中文输入法候选框错位 |
| 长文本粘贴 | 可粘贴大段代码/日志，界面折叠显示，发送给 AI 的仍是完整内容 |
| Claude Code 常用命令 | 支持 `/help`、`/new`、`/clear`、`/status`、`/cost`、`/compact`、`/resume`、`/model`、`/exit` |
| no-tui 思考流 | 默认显示可读思考摘要，不再把完整 JSON action 当思考内容刷屏 |
| 新运维工具 | `target_health_summary`、`disk_usage`、`file_tail`、`login_audit`、`service_units` |

## 1. 快速开始

### 1.1 安装与编译

```bash
git clone <repo>
cd DeepSentry-deepagent
go build -o deepsentry ./cmd/main.go ./cmd/usage.go ./cmd/survey_compat.go ./cmd/console_other.go
# Windows: 使用 console_windows.go 替代 console_other.go
```

### 1.2 首次运行（配置向导）

```bash
./deepsentry --init
```

向导会引导你配置：

| 步骤 | 说明 |
|------|------|
| AI 服务商 | DeepSeek / Qwen / 电信星辰 TeleAI / 腾讯混元 / OpenAI / Claude / Gemini / MiMo / Ollama 等 |
| API 地址 | 支持只填 `/v1`，自动补全 `chat/completions` |
| 模型名称 | 如 `mimo-v2.5-pro`、`deepseek-v4-pro`、`qwen-plus`、`hunyuan-turbos-latest` |
| API Key | 本地 Ollama 可留空 |
| 最大轮数 | 默认 30，防止 Agent 死循环 |
| 管理模式 | 默认本地；Windows cmd 方向键无效时可用 Tab / Shift+Tab 切换 |
| 连接信息 | 根据模式录入 SSH/Telnet/FTP 或多台 `targets[]` |

配置保存至 `./config.yaml` 或 `~/.deepsentry/config.yaml`。

**管理模式说明：**

| 模式 | 结果 |
|------|------|
| 本地任务 | 清空远程连接字段，所有工具在控制端本机执行 |
| 远程管理 1 台服务器 | 写入旧版兼容字段：`target_protocol`、`ssh_*`、`telnet_*`、`ftp_*` |
| 远程管理多台服务器 | 写入 `targets[]`，主进程保持本地控制端模式，由 Fleet/subagent 逐台连接 |

### 1.3 推荐：TUI 模式（类 Claude Code）

```bash
./deepsentry --tui
./deepsentry --tui "排查目标机内存与监听端口"
./deepsentry --plan "配置每 10 分钟 CPU 监控并通知"
./deepsentry --tui -c /path/to/config.yaml --resume session_xxx
```

**TUI 界面说明：**

```

│ ┌──────────────────────────────────────┐                   │

│ └──────────────────────────────────────┘                   │
│     └ MemTotal: 1725484 kB ...                             │
├─ SSH -> 8.x.x.x:2222 │ 本地/无活动目标 │ Step 1/30 │ 10:42:05 ┤

│ follow up, Enter to send...                                 │

```

| 按键 | 功能 |
|------|------|
| `↑` / `↓` / `j` / `k` | 滚动 Agent 活动日志 |
| `PgUp` / `PgDown` | 翻页 |
| `Tab` | 聚焦底部输入框 |
| `Enter` | 发送任务 / 后续指令 |
| `Shift+Enter` / `Alt+Enter` / `Ctrl+J` | 输入换行（取决于终端是否上报） |
| `Ctrl+U` | 清空当前输入 |

| `q` / `Ctrl+C` | 退出 TUI |

**长文本粘贴：** 直接粘贴 1000 行代码/日志时，输入框只显示折叠摘要，提交给 AI 的仍是完整内容。  
**中文输入法：** 当前版本把真实输入行固定为最后一行，候选框会贴近输入行；如果仍显示旧样式，请确认启动的是重新编译后的同一个二进制。

### 1.3.1 TUI 斜杠命令

输入 `/` 会显示内置命令联想；继续输入前缀可过滤，例如 `/c` 会优先联想到 `/clear`。`↑↓` 切换候选，`Tab` 补全，`Enter` 执行完整命令。

| 命令 | 功能 |
|------|------|
| `/help` | 查看快捷键和命令 |
| `/new` | 开启全新任务/会话；`/new <任务>` 会立即启动 |
| `/restart` | 等同 `/new` |
| `/clear` | 清空当前屏幕日志 |
| `/status` | 查看连接、模型、运行状态 |
| `/cost` | 查看真实 token 用量；模型未返回 usage 时显示估算值 |
| `/compact` | 记录上下文整理提示 |
| `/resume` | 查看恢复会话的命令提示 |
| `/model` | 查看当前模型 |
| `/sessions` | 列出 checkpoint 会话 |
| `/config` | 显示配置摘要 |
| `/exit` / `/quit` | 退出 TUI |

### 1.3.2 多轮对话

任务完成后，底部输入框自动激活。可直接输入后续指令（如「再查一下 443 端口」），Agent 会在同一上下文中继续执行。

### 1.3.3 子 Agent 面板

委派子 Agent 时显示黄色边框面板；结果默认折叠，按 `e` 展开完整输出。

Fleet 多目标任务会显示 target 状态，包括 pending/running/ok/error、协议、主机和最后动作。

### 1.3.4 会话选择器

```bash
./deepsentry --tui --pick-session
```

图形化列出 checkpoint 会话（ID、步数、时间），Enter 恢复，N 新建。

### 1.3.5 Benchmark 可视化 TUI

```bash
go run ./cmd/benchmark/ -c config.yaml --tui
```

实时显示 61 个场景进度条与维度得分雷达。

### 1.4 经典 CLI 模式

```bash
./deepsentry --no-tui "分析 auth.log 中的暴力破解 IP"
./deepsentry --no-tui -c config.yaml --batch -y "自动巡检"
./deepsentry --no-tui --json --task "mem_info + port_listen 摘要"
./deepsentry --webshell --task "读取最近登录日志"
```

说明：

- 默认运行会进入 TUI；脚本、WebShell、CI 建议显式加 `--no-tui`。
- `--no-tui` 普通输出会显示 AI 思考摘要，不会把完整 JSON action 作为思考内容刷屏。
- `--json` 输出 JSONL 事件，适合外部程序消费。
- `--webshell` 会提交后台任务并立即返回报告/进度路径：后台子进程启用 `--no-tui --batch -y`，命令 stdout/stderr 会像 fscan 一样逐行写入 `reports/webshell_progress_<timestamp>.log`，完整结果仍写入报告并保留给 Agent；固定索引在 `reports/latest_webshell.txt`。
- WebShell 补充信息恢复示例：`./deepsentry --webshell --resume session_xxx --task "Webhook 是 https://..."`

交互输入时会显示**示例建议**（Tab/方向键可选取）：

- 排查目标机内存与监听端口
- 分析 SSH 登录失败记录
- 审计网络暴露面与异常连接
- 在 Web 目录狩猎 webshell

## 3. 工作模式

### 3.1 本地模式、远程单机、Fleet 多机

| 模式 | 触发条件 | 工具视角 |
|------|----------|----------|
| **本地** | 单机连接字段为空，且没有指定远程目标任务 | 所有操作在控制端本机 |
| **远程单机** | 配置 `target_protocol` + 对应 `ssh_*` / `telnet_*` / `ftp_*` | 🎯 目标机 / 💻 控制端 双视角 |
| **Fleet 多机** | 配置 `targets[]` | 主进程本地控制端，按 selector 为每台目标创建连接 |

- **目标机工具**（读 `/proc`、SFTP）：`mem_info`、`port_listen`、`process_list`、`read_file`…
- **控制端工具**（从 DeepSentry 进程发起）：`ping`、`dns_lookup`、`http_probe`、`nmap_scan`…

### 3.2 Fleet 多目标配置

```yaml
targets:
  - name: web-01
    protocol: ssh
    host: 10.0.0.11:22
    user: root
    password: ""
    key_path: ~/.ssh/id_rsa
    tags: ["prod", "web"]

  - name: legacy-router
    protocol: telnet
    host: 10.0.0.2:23
    user: admin
    password: admin
    telnet_prompt: ">"
    tags: ["legacy", "network"]

  - name: ftp-backup
    protocol: ftp
    host: 10.0.0.50:21
    user: backup
    password: xxx
    tags: ["backup"]
```

selector 支持：

| selector | 命中规则 |
|----------|----------|
| `all` | 全部目标 |
| `web-01` | 按 name |
| `10.0.0.11` | 按 host 包含 |
| `ssh` / `telnet` / `ftp` | 按协议 |
| `prod` / `legacy` | 按 tag |

### 3.3 Fleet 与 target-aware 子 Agent

| 方式 | 适合场景 | 示例 |
|------|----------|------|
| `fleet_inventory` | 查看目标清单 | “列出所有 prod 目标” |
| `fleet_exec` | SSH/Telnet 批量执行只读巡检命令 | “对 prod 执行 uptime 和 df -h” |
| `fleet_file` | SSH/Telnet/FTP 文件操作 | “读取所有目标 /var/log/auth.log 末尾 100 行” |
| `task + target_selector` | 每台机器独立分析、最后汇总 | “让 log-analyst 分析所有 prod 机器登录日志” |

`task + target_selector` 示例：

```json
{
  "action": "task",
  "task_name": "log-analyst",
  "target_selector": "prod",
  "task_prompt": "审计最近 24 小时 SSH 登录日志，列出高风险 IP 和成功登录用户"
}
```

注意：

- 每个目标会创建独立 executor，不共享全局连接，避免多机并发串目标。
- 低风险只读任务可自动并发；写入、批量修改、高风险命令需要确认。
- 已配置 targets 时不要让 Agent 裸跑 `ssh/scp/sftp root@host`；这些命令不会读取 DeepSentry 配置里的密码/私钥，请使用 `fleet_exec`、`fleet_file` 或 `target_selector`。
- FTP 目标不能执行 shell，因此 `fleet_exec` 会拒绝 FTP；请使用 `fleet_file` 做 `ls/read/download/upload`。

### 3.4 Agent 动作协议

Agent 每步输出 JSON action，支持：

| action | 用途 |
|--------|------|
| `tool` | 调用内置 Go 原生工具 |
| `execute` | Shell 命令（高风险需确认） |
| `task` | 委派子 Agent；可带 `target_selector` 分解多目标任务 |
| `load_skill` | 加载安全 Skill |
| `todo` | 任务规划 |
| `ask_user` | 缺少关键信息时暂停等待用户补充 |
| `read_file` / `grep` / `ls` / `glob` / `edit_file` | 文件操作 |
| `remember` / `forget` | 跨会话记忆 |
| `finish` | 完成任务 |

### 3.5 跨会话 Memory

DeepSentry 默认内置一份 `builtin://AGENTS.md`，随单二进制一起编译，不需要额外携带文件。

可选扩展：

- `~/.deepsentry/AGENTS.md`：用户级长期偏好
- `.deepsentry/AGENTS.md`：项目级规范
- `~/.deepsentry/memory/store.json`：`remember` / `forget` 写入的结构化记忆

Memory 会自动注入 Agent 上下文；明显的 API Key、密码、Token、私钥等敏感凭证会被拒绝保存。

## 4. 安全应急场景指南

### 4.1 内存 / CPU 异常

```
排查目标机内存使用，列出前 20 个进程，给出结论
```

推荐工具链：`mem_info` → `process_list` → `finish`

### 4.2 网络暴露面 / 异常连接

```
审计目标机监听端口和 established 连接，识别可疑出站
```

推荐：`port_listen` → `net_connections` → `arp_table`

可先 `load_skill network-recon`。

### 4.3 日志取证

```
分析 /var/log/auth.log 中 Failed password，统计 Top 攻击 IP
```

推荐：`load_skill log-analysis` → `read_log` / `grep` → `finish`

### 4.4 Webshell 狩猎

```
在 /var/www 目录查找可疑 PHP 文件（eval/base64_decode）
```

推荐：`load_skill webshell-hunt` → `glob` + `file_strings` + `file_ident`

### 4.5 漏洞 / 基线核查

```
检查空口令账户、SUID 文件、crontab 后门
```

推荐：`load_skill vuln-scan` → 委派 `vuln-scanner` 子 Agent

### 4.6 本地 + 远程联动

```
从控制端 http_probe 探测目标 80 端口，同时在目标机 port_listen 核对
```

## 6. 配置详解

参考 `config.example.yaml`：

```yaml
provider: mimo
api_url: https://your-api/v1
model_name: mimo-v2.5-pro
api_key: sk-xxx

target_protocol: ssh
ssh_host: 8.x.x.x:2222
ssh_user: root
ssh_password: xxx

# 多机模式使用 targets[]；单机字段可留空
targets:
  - name: web-01
    protocol: ssh
    host: 10.0.0.11:22
    user: root
    tags: ["prod", "web"]

max_steps: 30
llm_retries: 3
ssh_command_timeout: 90
use_native_tools: false
```

**环境变量覆盖：** `DEEPSENTRY_API_KEY`、`DEEPSENTRY_SSH_HOST` 等（前缀 `DEEPSENTRY_`）。

### 6.1 基础运维工具速查

| 工具 | 用途 |
|------|------|
| `target_health_summary` | 跨平台健康摘要：系统、负载、磁盘、内存、Top 进程 |
| `disk_usage` | Linux/macOS/Windows 磁盘使用情况 |
| `file_tail` | 大日志尾部读取 |
| `document_parse` | 原生解析 PDF/Word DOCX/Excel XLSX/XLS/CSV/RTF，提取文本、表格和元信息 |
| `login_audit` | SSH/auth.log/secure/last 或 Windows Security 登录摘要 |
| `service_units` | systemd 服务或 Windows Service 列表 |
| `process_list` | Linux `/proc`，Windows 自动降级 `tasklist` |
| `net_connections` | Linux `/proc/net`，Windows 自动降级 `netstat -ano` |
| `pcap_analyze` | gopacket 离线解析 pcap：协议统计、会话、DNS/HTTP/TLS SNI/SMB/NTLM 线索 |
| `tcp_forward` | 授权 TCP 端口映射，类似 iox/lcx 的本地转发，短生命周期 |
| `socks5_proxy` | 授权 SOCKS5 代理，类似 nps/iox 的本地代理能力，仅 CONNECT |
| `schedule_task` | 原生定时任务：解析“明天9点/每天/每周”等自然语言，定时巡检、生成报告并可发钉钉/飞书/邮件网关通知 |

## 7. 会话恢复（Checkpoint）


LLM 连续失败 3 次也会提示 checkpoint 路径。

## 9. 安全边界

| 规则 | 说明 |
|------|------|
| 受保护路径 | 禁止读 `config.yaml`、`reports/` |
| Workspace 白名单 | `~/.deepsentry/workspace` 可读写 |
| 高风险确认 | `execute` 高危命令需 Y/N；`fleet_exec` / `fleet_file` 会按真实命令或文件动作动态判险 |
| 双视角隔离 | `ping`/`nmap` 不会误走目标机 shell |

## 10. 故障排查

| 现象 | 处理 |
|------|------|
| SSH 连接失败 | 检查端口（如 2222 非 22）、`-init` 重配密码 |
| LLM 429/超时 | 增大 `llm_retries`，检查 API 配额 |
| Agent 空转 | 减小任务粒度，或 `--resume` 继续 |
| TUI/向导乱码或图标变方块 | Windows `cmd.exe` 和部分国产化终端会自动降级为 ASCII 图标但保留颜色；也可手动设置 `DEEPSENTRY_PLAIN=1` 或 `DEEPSENTRY_ASCII=1`。Windows Terminal/PowerShell 7 可用 `DEEPSENTRY_FANCY=1` 开启漂亮图标 |
| 需要纯无色输出 | 使用 `--no-color`，或设置 `NO_COLOR=1` / `DEEPSENTRY_NO_COLOR=1` |
| 中文输入法候选框错位 | 重新编译并确认启动的是新产物；当前版本输入行固定在最后一行 |
| 编译后界面还是旧的 | 可能启动了旧二进制。推荐 `bash build.sh && cd build && ./deepsentry -c config.yaml`，或根目录 `go build -o deepsentry ./cmd/main.go ./cmd/usage.go ./cmd/survey_compat.go ./cmd/console_other.go && ./deepsentry -c config.yaml` |
| process_list 慢 | 已优化为限量读取；可调小 `limit` 参数 |
| FTP 批量命令失败 | FTP 不支持 shell，请使用 `fleet_file` 的文件动作 |

## 12. 典型工作流示例
